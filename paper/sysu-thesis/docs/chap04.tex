\chapter{方法}
\label{cha:method}

\section{方法概述}
\label{sec:method-overview}

本文方法的核心思想是：在冻结模型参数的前提下，将边缘模型的高频感知能力、云端模型的低频纠错能力与外部知识的可复用性组织为统一闭环。设边缘模型为 $f_{\theta_e}$，云端模型为 $g_{\theta_c}$，其中 $\theta_e$ 与 $\theta_c$ 在实验期间均保持不变。对于输入样本 $x=(I,q)$，系统首先执行边缘感知，随后依据不确定性与任务规则决定是否触发约束化再感知或云端纠错。若纠错结果具有稳定价值，则进一步将其抽象为结构化技能并写入技能库 $\mathcal{S}$。

因此，本文讨论的“适配”并非参数层面的再训练，而是推理路径与外部知识的动态重组。该设计适合长尾场景中“快速补救、逐步积累、谨慎复用”的部署需求。其核心逻辑在于：长尾失效通常只集中在一小部分高风险样本中，因此系统应把更强的计算资源和更复杂的知识调用限制在这一小部分样本上，而不是无差别扩大全部样本的推理代价。

\section{边缘端结构化感知}
\label{sec:method-edge}

边缘端承担高频主链路。给定图像 $I$ 和问题 $q$，边缘模型输出基础场景理解结果
\begin{equation}
  z_e = f_{\theta_e}(I,q)=\{\hat{y}_e,p_e\},
\end{equation}
其中 $\hat{y}_e$ 为结构化感知结果，$p_e$ 为候选标签分布。本文将结构化感知结果设计为由问答结论、对象关系、局部区域描述、动作建议和必要元信息组成的统一表示，以便后续模块能够直接使用，而不必重新从自由文本中提取证据。

这一设计具有两方面作用。其一，它避免将系统决策建立在单句自然语言答案之上，从而提高了误差分析和路由判断的可操作性。其二，它为后续的外部知识调用提供了清晰接口，使得结构化技能能够围绕关注区域、问题分解和输出约束对边缘结果进行修正。对于自动驾驶任务而言，这种结构化接口尤其重要，因为系统不仅需要知道“答案是否正确”，还需要知道“错误最可能源于哪个证据缺口”。

从方法动机看，边缘端之所以首先输出结构化结果，而不是直接输出最终自由文本答案，是因为本文的后续模块均依赖统一的中间表示。如果边缘模型只输出一句自然语言描述，则路由决策难以识别局部不确定性，云端纠错也难以判断究竟应重看哪些区域，结构化技能更无法稳定复用。这一设计保证了整个系统在推理过程中具有统一的信息接口。

\section{不确定性驱动的选择性路由}
\label{sec:method-routing}

为了避免无差别地调用云端模型，本文引入不确定性驱动的选择性路由机制。对于边缘模型给出的候选分布 $p_e$，本文使用熵
\begin{equation}
  u(x)=H(p_e)=-\sum_{i=1}^{K} p_i \log p_i
\end{equation}
作为不确定性代理量。若 $u(x)$ 超过阈值，或当前结果触发风险规则，则系统认为该样本更可能属于边缘模型的脆弱区域，需要进一步处理。

在此基础上，本文将推理流程划分为三层。第一层为边缘直接输出，即当不确定性较低且未命中风险条件时，保留边缘感知结果作为最终输出。第二层为约束化再感知，即当系统检索到与当前样本高度相关的结构化技能时，优先利用该技能对边缘模型进行有约束的再次观察。第三层为云端纠错，即当约束化再感知仍无法消除不确定性，或样本命中高风险失效模式时，调用云端模型执行更强的图像级纠错。

这一机制的关键在于，云端资源并非默认主链路，而是一种在风险受控前提下启用的补救资源。其设计动机来自两个观察：一方面，自动驾驶长尾错误往往呈现集中分布，系统没有必要为所有样本都承担大模型代价；另一方面，高风险样本若完全依赖边缘输出，又容易在远距离目标和局部关键证据缺失时出现系统性误判。因此，选择性路由本质上是在“云端算力稀缺”与“长尾风险集中”之间寻找折中。

在工程实现上，本文的路由机制不追求复杂控制器，而强调可解释性与可调节性。路由决策由不确定性估计与规则风险共同驱动，这使系统能够直接回答“为什么这个样本被送往云端”，从而为部署评估和风险审计提供依据。

\section{云端纠错与图像级再感知}
\label{sec:method-cloud}

云端模块的设计目标是纠正边缘模型在关键长尾样本上的系统性失效。与仅围绕边缘输出进行文本反思的方案不同，本文采用图像级再感知策略，即在触发后仍以原始图像和问题为主要依据执行纠错，同时将边缘结果视为辅助上下文，而非强约束前提。设云端纠错结果为
\begin{equation}
  y_c = g_{\theta_c}(I,q,\hat{y}_e),
\end{equation}
则最终输出可以写为
\begin{equation}
  \hat{y}(x)=
  \begin{cases}
    \hat{y}_e, & r(x)=0,\\
    y_c, & r(x)=1 \land \text{边缘结果未被可靠修正},\\
    \tilde{y}_e, & r(x)=1 \land \text{约束化再感知已满足稳定性条件},
  \end{cases}
\end{equation}
其中 $\tilde{y}_e$ 表示在外部知识约束下获得的边缘再感知结果。

采用图像级再感知的动机在于，自动驾驶中的远距离失效常源于视觉证据本身的缺失或误读，而不仅是语言层面的表达偏差。若纠错过程过度依赖边缘模型已有解释，云端模型容易被错误语义锚定，从而削弱补救效果。换言之，真正需要修复的往往不是语言描述本身，而是视觉观察起点。

本文因此将云端设计为“重新观察原始图像的强模型”，而不是“对边缘文本答案进行二次润色的语言模型”。这一设计使云端纠错与结构化技能机制形成互补：前者提供高上限补救能力，后者提供低代价经验复用能力。

\section{结构化技能沉淀机制}
\label{sec:method-skill}

本文将结构化技能定义为可检索、可约束、可复用的外部知识单元。对于一次成功纠错样本，其关键信息被抽象为
\begin{equation}
  s=(c_s,\rho_s,\psi_s,\omega_s,b_s),
\end{equation}
其中 $c_s$ 表示触发条件集合，$\rho_s$ 表示需要优先关注的区域约束，$\psi_s$ 表示问题分解或观察顺序，$\omega_s$ 表示输出约束，$b_s$ 表示在证据不足时的安全回退策略。

该表示方式的意义在于，将一次性的纠错结果转化为可复用的外部知识，而不是让经验仅停留在当前样本。与一般的自由文本提示不同，结构化技能强调输入条件、推理路径与输出边界的显式化，因此更适合自动驾驶这类安全关键场景中的知识复用。

从研究动机看，本文之所以不直接缓存“问题—答案”对，而是引入包含触发条件、关注区域和回退策略的结构化技能，是因为自动驾驶中的经验复用必须回答三个问题：什么条件下应当调用该经验、调用后应重点看哪里、当证据不足时应如何避免错误扩散。只有将这三个问题显式写入知识表示，经验复用才可能既带来收益又保持安全边界。

\section{技能检索与约束化再感知}
\label{sec:method-match}

为了在不引入大规模误匹配的前提下使用已有知识，本文对技能检索采用“语义相似度 + 上下文一致性”的联合评分策略。对于输入样本 $x$ 和候选技能 $s$，定义匹配分数为
\begin{equation}
  \mathrm{Score}(x,s)=
  \lambda_1 \mathrm{Sim}_{\mathrm{sem}}(x,s)+
  \lambda_2 \mathrm{Match}_{\mathrm{tag}}(x,s)+
  \lambda_3 \mathrm{Match}_{\mathrm{ctx}}(x,s),
\end{equation}
其中 $\mathrm{Sim}_{\mathrm{sem}}$ 表示语义相似度，$\mathrm{Match}_{\mathrm{tag}}$ 表示触发条件的一致程度，$\mathrm{Match}_{\mathrm{ctx}}$ 表示天气、区域、任务类型等上下文匹配程度。只有当评分超过阈值时，技能才会参与推理。

在被激活后，结构化技能不会直接替代模型输出，而是以约束的形式指导再次观察。这样一来，系统既能复用历史经验，又能避免将错误知识硬性注入当前样本。

进一步地，约束化再感知的核心并不是“强行让模型接受某个答案”，而是“缩小再次观察的自由度并明确输出边界”。例如，技能可以要求模型优先检查远处人行道区域、先判断目标存在性再判断数量、若证据不足则保持保守输出等。这种设计比直接提供模板答案更适合安全关键场景，因为它强调的是观察顺序和约束边界，而不是结论的先验灌输。

\section{策略目标与推理算法}
\label{sec:method-objective}

由于本文不对主模型参数进行更新，因此方法中的“优化”主要体现在推理策略层面。可以将单样本策略效用写为
\begin{equation}
  \mathcal{U}(x)=\mathbb{I}[\hat{y}(x)=y^\star]-\lambda_c \mathbb{I}[r(x)=1]-\lambda_l \tilde{t}(x),
\end{equation}
其中 $\mathbb{I}[\cdot]$ 为指示函数，$y^\star$ 为真实标签，$\lambda_c$ 与 $\lambda_l$ 分别刻画云调用代价和时延代价的权重，$\tilde{t}(x)$ 表示归一化后的推理时延。该目标并不通过梯度下降直接求解，而是作为路由阈值、风险规则和技能调用策略的设计依据。

为了便于说明推理流程，算法~\ref{alg:inference} 给出本文系统的单样本执行过程。

\begin{algorithm}[htbp]
  \caption{免训练分层边缘云协同推理流程}
  \label{alg:inference}
  \KwIn{输入样本 $x=(I,q)$，边缘模型 $f_{\theta_e}$，云端模型 $g_{\theta_c}$，技能库 $\mathcal{S}$}
  \KwOut{最终结构化结果 $\hat{y}(x)$}
  计算边缘结果 $z_e=\{\hat{y}_e,p_e\}\leftarrow f_{\theta_e}(I,q)$\;
  计算不确定性 $u(x)$ 并评估风险规则 $\phi(x)$\;
  \uIf{$u(x)<\tau_h$ 且 $\phi(x)=0$}{
    返回 $\hat{y}_e$\;
  }
  检索与 $x$ 匹配的候选技能集合 $\mathcal{S}_x \subseteq \mathcal{S}$\;
  \uIf{$\mathcal{S}_x \neq \varnothing$}{
    在技能约束下执行边缘再感知，得到 $\tilde{y}_e$\;
    \uIf{$\tilde{y}_e$ 满足稳定性条件}{
      返回 $\tilde{y}_e$\;
    }
  }
  执行云端图像级纠错 $y_c\leftarrow g_{\theta_c}(I,q,\hat{y}_e)$\;
  \If{当前样本满足适配评估协议下的沉淀条件}{
    将纠错经验抽象为结构化技能并写入 $\mathcal{S}$\;
  }
  返回 $y_c$\;
\end{algorithm}

\section{计算特性、适用条件与边界}
\label{sec:method-complexity}

从计算结构上看，本文方法的单样本开销可分解为三部分：边缘推理开销、技能检索开销和云端纠错开销。若边缘模型计算成本记为 $C_e$，云端模型计算成本记为 $C_c$，技能检索成本记为 $C_s$，则系统的期望开销可近似写为
\begin{equation}
  \mathbb{E}[C]\approx C_e + P_{\mathrm{skill}} C_s + P_{\mathrm{cloud}} C_c ,
\end{equation}
其中 $P_{\mathrm{skill}}$ 和 $P_{\mathrm{cloud}}$ 分别表示技能激活概率和云端调用概率。在当前主实验中，技能持久化未参与主链路，因此主结果主要由 $C_e + P_{\mathrm{cloud}} C_c$ 决定。若后续采用向量索引或近邻检索结构，技能检索开销可以进一步控制在较低水平；若采用线性扫描，则其开销将随技能库规模增长而增加[需补充检索索引实现细节]。

本文方法适用于以下条件：一是系统存在明确的高频基础链路与低频高能力补救链路；二是错误呈现一定的结构性集中，而非完全随机分布；三是任务允许输出结构化中间结果，以支持知识抽取和风险审计。同时，本文方法也存在清晰边界：若边缘模型基础能力过弱，则选择性协同可能退化为频繁云调用；若技能检索精度不足，则知识复用反而可能引入伤害；若真实分布与构建技能时的场景差异过大，结构化技能的收益也会迅速衰减。

\section{双协议实验设计}
\label{sec:method-protocol}

为了保证主基准结论与适配机制分析之间的边界清晰，本文采用两类实验协议。

\subsection{基准忠实协议}

该协议用于验证选择性混合路由本身是否有效。其特点是：仅比较边缘单独推理、云端单独推理和选择性混合路由三种在线配置；不允许主基准样本持续写入结构化技能库；所有结论均限定在清洗后的共享样本集合上。该协议回答的是系统在线路由是否有效，而不是外部知识是否已经稳定可学。

\subsection{适配评估协议}

该协议用于分析结构化技能沉淀与复用的收益和风险。其任务包括：检验纠错经验能否被稳定抽象为结构化知识；检验这些知识在相似场景中的复用效果；分析规模扩大后可能出现的检索污染、跨场景失配与错误放大。该协议不与主基准结果混写，因此不会夸大方法的证据强度。

\section{本章小结}
\label{sec:method-summary}

本章从边缘感知、路由决策、云端纠错、结构化技能沉淀、约束化再感知以及计算特性分析六个层面介绍了本文方法。总体而言，本文方法不依赖参数更新，而是通过推理路径设计与外部知识组织实现测试时增强，这也是其区别于传统再训练方案的关键所在。
