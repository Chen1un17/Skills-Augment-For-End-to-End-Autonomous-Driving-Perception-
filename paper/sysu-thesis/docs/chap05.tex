\chapter{实验设计、结果与分析}
\label{cha:experiment}

\section{实验目标与证据边界}
\label{sec:exp-goal}

本章实验围绕三个层级的结论展开。第一层为主实验 A，验证 selective cloud routing 在 benchmark-faithful 协议下是否能够提升长尾感知效果；第二层为主实验 B，验证 skill refinement 作为独立 test-time adaptation 机制是否成立；第三层为真实数据 pilot，分析系统在外部效度上的局限。三层实验分别对应强结论、机制结论与观察结论，彼此不混写、不相互替代。

需要特别说明的是，本文使用的本地实验 artifact 仅作为内部证据来源，用于保证数值可追溯；正文中的参考文献全部来自公开文献，不使用任何本地日志、Markdown 报告、CSV 或 JSON 文件作为 \verb|\cite| 条目。

\section{实验设置}
\label{sec:exp-setup}

\subsection{数据集与主任务}

主定量实验使用 DTPQA 的 \texttt{DTP-Synth / Category 1} clean 50-sample shared subset。该子集聚焦 pedestrian presence 问题，能够直接暴露边缘模型在远距离正样本上的假阴性失败模式\cite{theodoridis2025dtpqa,theodoridis2025evaluating}。此外，本文还使用 \texttt{DTP-Real / Category 1} pilot 进行真实数据局限性分析，并使用已有系统 artifact 组织独立 skill 复用实验。

\subsection{模型与执行模式}

本文比较三种执行模式：

\begin{enumerate}
  \item \texttt{edge\_only}：仅使用边缘模型完成单次感知；
  \item \texttt{cloud\_only}：使用云端模型直接完成同一任务；
  \item \texttt{hybrid}：以边缘模型为主链路，仅在满足条件时调用云端。
\end{enumerate}

在当前主实验中，\texttt{hybrid} 的反思触发率为 18\%，对应 50 个样本中的 9 个 direct cloud re-perception 调用。由于 benchmark-faithful 协议显式禁用了 benchmark-derived skill 持久化，因此主实验中的技能匹配率为 0。

\subsection{评测指标}

为同时衡量效果与系统代价，本文采用如下指标：

\begin{enumerate}
  \item \textbf{任务效果指标}：Exact Match Accuracy、Yes Recall、No Specificity、Far Accuracy；
  \item \textbf{系统代价指标}：Mean/P50/P95 Latency、Reflection Rate、Cloud-call Reduction；
  \item \textbf{系统机制指标}：Hybrid Rescue Rate、Hybrid Harm Rate、Reflection Precision；
  \item \textbf{独立适配指标}：复用前后准确率变化、修复样本数、伤害样本数、技能调用比例。
\end{enumerate}

\subsection{时延与系统代价}

准确率提升只有在系统代价可接受时才具有部署意义。图~\ref{fig:latency} 给出了三种模式的平均时延、中位数时延与 P95 时延。

\begin{figure}[htbp]
  \centering
  \includegraphics[width=0.90\textwidth]{image/thesis/fig_latency.pdf}
  \bicaption{三种模式的时延分布统计}{Latency statistics of the three execution modes}
  \label{fig:latency}
\end{figure}

由图~\ref{fig:latency} 可见，\texttt{cloud\_only} 的平均与长尾时延都远高于 \texttt{edge\_only}；\texttt{hybrid} 虽然引入了部分高时延样本，但其平均时延仍显著低于 \texttt{cloud\_only}。在当前 clean subset 上，\texttt{hybrid} 相比 \texttt{cloud\_only} 实现了 82\% 的云调用削减和 74.9\% 的平均时延下降，这说明其收益并非来自“大规模转向云端”。

\subsection{系统级收益与伤害分析}

为了直接衡量 \texttt{hybrid} 是否真正发挥了系统价值，本文进一步统计其 rescue rate、harm rate 与云调用削减比，如图~\ref{fig:system-effectiveness} 所示。

\begin{figure}[htbp]
  \centering
  \includegraphics[width=0.85\textwidth]{image/thesis/fig_system_effectiveness.pdf}
  \bicaption{Hybrid 路由的系统级收益}{System-level gains of the hybrid route}
  \label{fig:system-effectiveness}
\end{figure}

在 50 个样本中，\texttt{edge\_only} 共出现 5 个错误，\texttt{hybrid} 修复了其中 4 个，对应 80\% 的 rescue rate；与此同时，\texttt{hybrid} 没有引入新的正确样本退化，harm rate 为 0。这一结果十分关键，因为安全关键系统不能以“修对一部分、破坏另一部分”为代价获取收益。本文的主实验表明，当前路由策略能够在边缘错误最集中的 far 样本上产生较高收益，同时维持保守性。

\section{主实验 B：独立 Skill Refinement 实验}
\label{sec:exp-skill}

\subsection{为何将 Skill 实验与主 benchmark 分离}

如果将 benchmark-derived skill 直接写入持久化技能库，那么后续样本会受到历史知识污染，导致主 benchmark 结果被高估。为避免这一问题，本文在 DTPQA 主实验中显式关闭 skill persistence。因此，主 benchmark 证明的是 selective cloud routing 有效，而不是“大规模技能学习已经完成验证”。

为了回答 skill refinement 是否具备独立机制价值，本文额外整理了 adaptation-oriented 实验，分别观察正向闭环、小规模风险与扩展风险三个场景。其结果如图~\ref{fig:skill-adaptation} 所示。

\begin{figure}[htbp]
  \centering
  \includegraphics[width=0.95\textwidth]{image/thesis/fig_skill_adaptation.pdf}
  \bicaption{独立 skill 适配实验中的收益与风险}{Gains and risks in the independent skill adaptation experiments}
  \label{fig:skill-adaptation}
\end{figure}

\subsection{正向闭环与扩展风险}

独立适配实验显示，小规模闭环中 skill reuse 可以带来正向收益。以正向 pilot 为例，系统在 15 个样本上将准确率从 0.8667 提升到 0.9333，带来 6.67 个百分点增益，并修复了 1 个 far-distance 样本且未引入伤害样本。该结果说明，结构化技能作为测试时知识单元是可以工作的。

但当 skill 复用规模扩大到 209 个样本时，系统准确率从 0.8373 下降到 0.7799，出现 25 个伤害样本，仅修复 13 个样本。这意味着 skill 机制虽然有潜力，但当前版本仍然存在检索污染、约束过强和跨场景泛化不足等问题。因此，本文将 skill refinement 定位为独立机制结论，而不是把它写成对主 benchmark 的附赠优势。

\subsection{典型 Skill Artifact 分析}

在真实系统 artifact 中，本文已经观察到完整的结构化技能编译链路。典型 skill 包含前方 lead vehicle、路侧 pedestrian、右侧作业区域等触发标签，并通过 focus region 与 dynamic question tree 显式约束模型的再次观察顺序。这一现象说明，training-free 不等于系统“不学习”，而是将学习从权重空间迁移到了外部知识空间。

\section{真实数据 Pilot：局限性与外部效度}
\label{sec:exp-real}

真实数据 pilot 不作为主胜利结果，而作为外部效度与失败模式分析。当前 \texttt{DTP-Real / Category 1} pilot 的整体准确率为 0.4444，far-distance 子集准确率为 0，说明真实数据中的距离退化比 synth clean subset 更为显著。

\begin{figure}[htbp]
  \centering
  \includegraphics[width=0.88\textwidth]{image/thesis/fig_real_pilot_failure.pdf}
  \bicaption{真实数据 pilot 中的 far-collapse 与 false-negative bias}{Far-collapse and false-negative bias observed in the real-data pilot}
  \label{fig:real-pilot}
\end{figure}

图~\ref{fig:real-pilot} 反映出两个现象。其一，真实数据中的 far-distance 正样本更容易被预测为 \texttt{No}，表现为明显的 false-negative bias；其二，unknown 距离样本虽然在 pilot 中看似更稳定，但这并不代表系统真正具备真实世界鲁棒性，因为它们往往对应更易判断的负样本。

进一步地，真实数据 intervention quick pilot 表明：若针对 far-distance failure 进行定向介入，共享样本上的准确率可以从 0.50 提升到 0.625，far 子集准确率可从 0 提升到 0.75；但与此同时，unknown 子集准确率会从 1.0 降至 0.3333。该现象说明，系统对真实数据的适配仍存在明显的分布敏感性与交叉伤害风险。

\section{结果讨论}
\label{sec:exp-discussion}

综合本章实验，可以得到以下结论。

\begin{enumerate}
  \item 在 benchmark-faithful 协议下，\texttt{hybrid} 已经证明了 selective cloud routing 的系统有效性，尤其在 far-distance 样本上表现出明显收益。
  \item skill refinement 作为独立 test-time adaptation 机制具备工程潜力，但当前版本仍未达到“稳定可扩展”的程度，因此不能与主 benchmark 结论混写。
  \item 真实数据 pilot 显示 synth clean subset 仍低估了外部世界中的 failure severity，系统在真实分布下依然存在 far-collapse、false-negative bias 和跨日漂移风险。
\end{enumerate}

\section{本章小结}
\label{sec:exp-summary}

本章按照“主实验 A、主实验 B、真实数据 pilot”的顺序完成了证据收口。总体而言，本文已经给出一个结论边界清晰的实验叙事：主 benchmark 证明 selective cloud routing 有效，独立适配实验说明 structured skill 有潜力但有风险，真实数据 pilot 则提醒我们当前系统离普适部署仍有距离。
