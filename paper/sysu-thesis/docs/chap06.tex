\chapter{讨论、局限与结论}
\label{cha:conclusion}

\section{方法有效性的进一步解释}
\label{sec:discussion}

本文结果表明，在冻结模型参数的条件下，自动驾驶长尾场景中的感知增强并不必然依赖重新训练。对于本文关注的距离敏感任务，边缘模型的错误主要集中在远距离正样本上，而选择性混合路由恰恰针对这一集中失效区域引入低频云端补救，因此能够在有限云调用下获得明显收益。从系统视角看，方法有效性的关键不在于是否引入了更强模型，而在于更强模型是否以受约束、可解释、可审计的方式进入原有链路。

从方法机理上看，本文结果可以从三个层面加以理解。第一，边缘路径为大多数低风险样本提供了低代价基线，因此系统总体效率得以保持。第二，云端纠错只在风险更高、边缘更脆弱的样本上触发，使额外算力真正集中服务于关键失效区域。第三，结构化技能机制虽然尚未稳定成熟，但已经显示出把单次纠错经验外化为长期知识的可能性。正是这三个层面的协同，使本文方法能够在不修改主模型权重的前提下实现测试时增强。

\section{与现有研究范式的关系}
\label{sec:discussion-paradigm}

本文工作与现有研究范式既有联系，也有明显区别。与传统再训练或微调方法相比，本文强调通过推理路径重组和外部知识调用获得能力补偿，因此更适合需要快速迭代、难以及时回流重训练的部署场景。与单纯的大模型接入方案相比，本文并不否认更强模型的重要性，而是试图回答“更强模型应如何进入系统”这一问题。与通用智能体中的反思和记忆机制相比，本文则进一步强调知识表示的结构化、调用条件的受限化以及错误回退的显式化。

这种关系意味着，本文并不是对已有研究的完全替代，而更像是在自动驾驶部署语境下对多条研究路线进行系统重组。其价值主要体现在把模型能力、系统代价和知识复用放入同一分析框架中加以讨论。

\section{对真实部署场景的启示}
\label{sec:discussion-deploy}

从工程实践角度看，本文有两点启示。第一，自动驾驶中的部署优化不应只关注模型平均精度，而应重点关注高风险失效模式是否可被定向修复。对于长尾场景而言，少量关键错误往往比大量普通样本上的微小增益更值得系统投入额外资源。第二，经验积累不必完全依赖重新训练。对于更新周期长、验证成本高的自动驾驶系统而言，把经验先沉淀到可审计、可灰度发布的外部知识层，再逐步评估其是否值得进入模型参数空间，可能是一条更稳妥的部署路径。

\section{方法局限}
\label{sec:limitations}

尽管本文已经给出较为完整的系统框架和初步证据，但仍存在以下局限。

\begin{enumerate}
  \item 当前最强证据主要来自 DTPQA 的单一任务子集，尚未覆盖完整基准或更多开放语义驾驶任务，因此结论的外推范围仍有限。
  \item 本文尚未给出完整的统计显著性检验与多次重复实验结果，因而当前报告更适合作为目标任务上的有力证据，而非最终定论。
  \item 结构化技能虽然在小规模闭环中表现出正向收益，但在复用范围扩大后仍存在检索污染与错误放大问题，说明其稳定性仍不足。
  \item 当前“边缘”与“云端”更多体现为角色分层而非真实硬件切分，因此本文时延结果仍需在真实车端与网络条件下进一步验证。
  \item 真实数据试验暴露出明显的远距离崩塌与假阴性偏置，表明当前方法对真实分布变化仍然敏感。
\end{enumerate}

\section{未来工作}
\label{sec:future}

针对上述局限，后续研究可以从以下几个方向继续推进。

\begin{enumerate}
  \item 扩展到更多自动驾驶感知问题和更完整的 DTPQA 子任务，验证方法在多类型长尾场景中的普适性。
  \item 建立严格分离的技能构建集与技能评估集，系统度量知识复用的精度、覆盖率、伤害率与长期收益。
  \item 在真实边缘硬件与车云通信环境下重新测量端到端时延、带宽占用与故障恢复行为，补足部署证据。
  \item 引入更细粒度的区域感知与风险估计机制，以缓解真实场景中的远距离假阴性问题。
  \item 完善统计显著性检验和配对消融实验，进一步巩固当前观察到的机制解释。
\end{enumerate}

\section{结论}
\label{sec:conclusion}

本文围绕自动驾驶长尾场景中的部署导向感知问题，研究了在冻结模型参数条件下如何通过边缘云协同和外部知识沉淀提升系统鲁棒性。针对远距离目标漏判、局部证据利用不足和开放语义判断不稳定等关键挑战，本文提出了免训练分层边缘云协同感知框架。该框架以边缘模型承担高频主链路，以云端模型承担低频纠错，并通过结构化技能沉淀实现测试时知识外化。

实验结果表明，在 DTPQA 的目标子任务上，选择性混合路由能够以较低云调用比例显著提升远距离样本识别效果，并保持相对可控的系统代价；独立适配实验说明结构化技能具有正向机制潜力，但其稳定收益仍依赖更严格的检索和约束设计；真实数据试验则揭示了当前方法在外部世界中的明显边界，特别是远距离假阴性偏置仍然突出。

总体而言，本文的主要贡献并不在于提出一种更大的模型，而在于给出一种更符合自动驾驶部署约束的研究路径：在边缘实时性前提下，以选择性云协同完成定向补救，并以结构化外部知识积累长期经验。该路径为自动驾驶长尾场景中的免训练系统适配提供了可进一步扩展的研究基础，也为后续围绕部署稳健性、知识安全复用和真实车云协同验证的研究奠定了问题框架。
